Edició de la documentació

Títols dels menús​

Només per la documentació en anglès (OneVision i VisionCloud)​

Tots els títols dels menús de l'esquerra de la documentació estaran en format "Title case", amb majúscules inicials a les paraules importants. Per exemple:

  • Export a Dataset
  • Create Your Application
  • Delete the Database

Per la documentació en català (Dev Docs)​

Els títols seran en format normal, amb la primera lletra de la primer paraula en majúscules. Per exemple:

  • Edició de la documentació

Títols dels submenús​

Els submenús no han d'anar amb "Title case", sinó que han d'anar com una frase normal.

Això ho hem tret de com ho fa Docusaurus en la seva documentació, a més a més queda molt millor sense majúscules a cada paraula.

Format del text​

caution

No abusar dels efectes de text, icones ni formats especials de text, poden molestar la lectura.

  1. Utilitzar negreta per destacar un element important: Model. No abusar de les negretes ni altres efectes per ressaltar el text.
  1. Ressaltar amb negreta els elements d'una llista, si és el cas.
  1. Per les seccions del software utilitzar cursiva negreta i el símbol > (Models, Datasets, Projects, Administration): Models
  1. Per els tabs, o subseccions dintre d'un menú, utilitzar cursiva: elements
  1. Mostrar estil botó per botons (Deploy, Start training): Create Dataset
  1. Utilitzar links per referències a altres seccions de la documentació o links externs: [Etiqueta del link](link relatiu a la pàgina actual). Per exemple: [Model Types]( ../available-models/choosing-model-type). IMPORTANT: Si es posa l'extensió del document (.md) no funcionarà la compilació.

Caixes per ressaltar text​

Utilitzar caixes per ressaltar text important (utilitzar sense abusar).

:::info
Only organization administrators can create new projects.
:::

Possibles variants de caixes, en ordre d'importància del missatge:

ClauDescripció
:::noteCaixa en blanc. Informació poc important.
:::tipCaixa en verd. Per informació rellevant, o que volem ressaltar.
:::infoCaixa en blau. Per informació neutra.
:::cautionCaixa en groc/taronja. Per avisos importants.
:::dangerCaixa en vermell. Per accions perilloses (eliminar dataset, eliminar usuari, etc).

Si es posa un text al costat, la caixa pren el títol d'aquell text. Per exemple :::info Avaluació.

Utilitzar taules​

Perquè les taules quedin centrades, s'ha d'utilitzar aquest format:

<div style={{ display: 'flex', justifyContent: 'center' }}>
  <table>
    <thead>
      <tr>
        <th>Format</th>
        <th>Extensions</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>JPEG</td>
        <td>.jpg, .jpeg</td>
      </tr>
      <tr>
        <td>PNG</td>
        <td>.png</td>
      </tr>
      <tr>
        <td>TIFF</td>
        <td>.tif, .tiff</td>
      </tr>
    </tbody>
  </table>
</div>

Utilitzar imatges​

tip

Veure la següent secció sobre com fer captures de pantalla dels softwares OneVision i VisionCloud (per la documentació pública).

Un cop les imatges ja estan generades, s'han d'utilitzar d'una forma concreta perquè el format de presentació sigui consistent. El format actual centra les imatges en el cos de la pàgina, i deixa uns marges. A més a més, afegeix un peu d'imatge.

Per fer servir el format correcte, fer servir el següent codi HTML:

<figure>
    <img src={require("./img/projects-new-form.png").default}/>
    <figcaption>New project creation form.</figcaption>
</figure>
note

En versions anteriors es posava el text alternatiu alt al costat de src={} però ja no es fa, es pot eliminar
