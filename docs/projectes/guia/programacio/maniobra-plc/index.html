Comunicacions amb el PLC

La seqüència que permet la comunicació entre el PLC/Robot i la visió, també anomenada maniobra, es pot fer de moltes maneres segons el funcionament del sistema i el que vulgui el client. A continuació, es presenten alguns exemples.

Maniobra sense trigger ni confirmació​

Projecte referència: PR050 Hero.

Es tracta d'un sistema de rebuig en el que el trigger es fa amb una fotocèl·lula i els resultats s'escriuen a dos booleans de sortida, OK i NOK. En funció del resultat de la inspecció, un d'aquests dos booleans es posa a true durant un interval de temps determinat.

note

No es recomana aquesta maniobra ja que no té cap tipus de resposta per part del PLC.

Maniobra amb trigger i sense confirmació​

Projecte referència: PR101 Tadesan.

El PLC o robot envia una senyal per capturar una imatge amb un booleà d'entrada, trigger, i espera el resultat de la visió que s'escriu amb dos booleans de sortida, OK i NOK.

Seqüència:

  1. El PLC espera que la visió tingui desactivades les senyals de resultat OK i NOK abans d'activar la senyal trigger.
  2. El PLC activa la senyal trigger.
  3. La visió captura la imatge, la processa, escriu el resultat a les variables OK i NOK, i es queda esperant a que la senyal trigger torni a false.
  4. Quan el PLC llegeix els resultats, posa trigger a false.
  5. Quan la visió veu trigger a false les variables OK i NOK.
note

Aquesta és la sincronització més senzilla amb un PLC o robot.

Maniobra amb ready, trigger i sense confirmació​

Projecte referència: PR095 Tecnowys.

Molt semblant a l'anterior però amb una senyal ready que indica que el PLC o la visió están preparats per comunicar.

Seqüència:

  1. El PLC posarà la variable PLC_ready a true quan arribi una peça, i escriurà el codi de la peça a una altra variable.
  2. La visió copiarà el codi de la peça i posarà la variable visio_ready a true.
  3. El PLC farà el trigger a la càmera, activant la senyal trigger.
  4. La visió analitzarà les imatges, i escriurà els resultats.
  5. El PLC posarà la variable trigger a false quan llegeixi els resultats.
  6. Per últim, la visió esperarà a que la variable trigger estigui a false per fer reset dels resultats i començar un nou cicle.

Maniobra amb trigger i confirmació​

Projecte referència: PR094 Biele.

En aquesta maniobra, el PLC envia una senyal de trigger, que la visió confirma amb trigger_confirm. La visió confirma l'escriptura dels resultats amb image_processed i el PLC confirma la lectura amb image_processed_confirm. Tot això de manera seqüencial.

Seqüència:

  1. El PLC detecta l’arribada d’una peça i activa la senyal trigger.
  2. La visió rep el trigger d’entrada i copia el valor a trigger_confirm. Aquesta confirmació permet al PLC enviar la peça cap a la càmera.
  3. La visió captura una imatge inicia el processament. En finalitzar, activa la variable image_processed, indicant que l’anàlisi ha acabat. També escriu el resultat del processament.
  4. Quan el PLC llegeix el resultat, confirma la lectura copiant el valor de image_processed a la variable image_processed_confirm.
note

En el projecte amb Biele aquestes senyals són enters que es van incrementant, per lo que no és necessari fer un reset al final de la seqüència.

Maniobra amb trigger i resultats independents amb confirmació​

Projecte referència: PR102 Sertec.

Es tracta d'un sistema en què el trigger i l'escriptura de resultats són independents. Per als triggers, el PLC escriu un booleà d'entrada, trigger, i la visió respon amb la variable trigger_rebut. Per als resultats, la visió escriu un booleà resultats_escrits i els propis resultats, i el PLC respon amb resultats_llegits.

En els casos en que la seqüència de triggers i la de resultats són diferents, el PLC ha de guardar i enviar un identificador amb la senyal de trigger per tal de saber a quina imatge es corresponen els resultats llegits.

Seqüència de trigger:

  1. El PLC enviarà una senyal per fer trigger a la càmera amb la variable trigger. Juntament amb aquesta senyal s’enviarà la posició d’encoder.
  2. Quan el programa vegi la variable trigger a true, posarà a true la variable trigger_rebut.
  3. El PLC veurà la variable trigger_rebut a true i posarà trigger a false. Moment en que el programa posarà també trigger_rebut a false.

Seqüència de resultats:

  1. La visió esperarà a que la variable resultats_llegits estigui a false.
  2. La visió escriurà els resultats.
  3. El PLC llegirà els resultats quan siguin diferents al valor per defecte, i posarà la variable resultats_llegits a true.
  4. Quan la visió vegi resultats_llegits a true farà reset dels resultats.
  5. Quan el PLC vegi els resultats amb el seu valor per defecte, posarà resultats_llegits a false.
