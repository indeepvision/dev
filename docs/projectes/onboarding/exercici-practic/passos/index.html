Passos a seguir

A continuació s'exposa a grans trets els passos a seguir per completar aquest exercici. No s'explica tot de forma exhaustiva: fer servir aquest document com a guia o requisits mínims. Un cop completada la funcionalitat mínima, pots experimentar afegint més funcionalitat.

Importar projecte plantilla​

  • Descarregar el projecte plantilla des d'aquesta pàgina.
  • Importar el projecte i assignar-li aquest nom al importar-lo: showkit3_{el-teu-nom}.
note

Fem servir snake_case pels noms dels projectes.

✅ Objectius​

  • Importar el projecte plantilla i canviar el nom del projecte a l'importar.

Preparar setup​

  • Demanar la càmera del magatzem.
  • Aconseguir caixa amb les cubetes, hauria d'estar al taller o per l'oficina.
  • Afegir càmera al projecte i connectar-se a la càmera.
  • Assegurar que el flat dome està encès de forma fixa.
  • Les imatges es capturen sense la tapa de la caixa de cubetes.
  • Definir els paràmetres de la càmera per aconseguir imatges similars a les d'entrenament.
  • Setejar de forma fixa els paràmetres de la càmera a la seqüència initCam. Si no es fa, els paràmetres de la càmera no seran persistents.
  • Configurar la seqüència de captura d'imatges per fer software trigger. Vigilar no configurar adquisició en mode continu.
  • Poder executar el programa i que mostri les imatges (sense resultats de processament).

✅ Objectius​

  • Configurar les càmeres en la seqüència INIT.
  • Tenir el llum encès de forma fixa.
  • Fer fotos amb la càmera i aconseguir imatges similars a les utilitzades per entrenar.
  • Poder executar l'aplicació i que funcioni com un "mode vídeo", només mostrant les imatges.

Localitzar el centre de les cubetes​

  • Descarregar el model que localitza els centres. Aquest model es diu af2024_loc_containers i fem servir la versió 2.
  • Importar el model i assignar-lo a un brain amb nom segmCentres.
  • Configurar la seqüència run del projecte per fer foto a cada cicle.
  • Enviar Capture amb la imatge al display.
  • Afegir processament d'imatge per detectar el centre de les cubetes amb DL.
  • El model s'ha d'executar sobre tota la imatge RGB, havent redimensionat prèviament la imatge a 306x256.
  • Crea una Action amb nom mostrarCentreCubetes que mostri opcionalment la segmentació del centre de les cubetes.
  • Assigna a les segmentacions el nom segmCentreCubeta i mostra'ls sota el grup centresCubetes.
Fer servir setFrame()

Evitar fer servir les funcions Ve.scale() per escalar resultats d'inferència a la mida original de les imatges. Per fer això, el millor és fer servir la funció segm.setFrame() sobre la imatge resultat de la segmentació, amb un rectangle de la mida de la imatge original. Això permet dibuixar les segmentacions en la posició correcta i que els resultats de la funció Ve.findImageContours() surtin en la posició correcta.

note

Fem servir camelCase per les variables dintre OneVision, i els brains es fan servir com a variables quan es fa Brains.segmCentres.inference().

✅ Objectius​

  • Executar cicles d'adquisició-processament de forma contínua.
  • Mostrar la segmentació dels centres lligada a l'estat d'una Action.
  • Has de provar que si mous les cubetes o tota la caixa es desplacen els centres.

Delimitar les cubetes amb rectangles​

  • Descarregar el model que troba el contorn de les cubetes. Aquest model es diu af2024_s3_area_vertical i fem servir la versió 1.
  • Importar el model i assignar-lo a un brain amb nom segmCubeta.
  • El model s'ha d'executar una vegada per cada cubeta.
  • El model s'ha d'executar sobre un crop RGB de 800x1100 px centrat en cadascun dels centres trobats en el pas anterior.
  • Aquest crop s'ha de redimensinonar a 400x550 abans de fer la inferència.
  • Per transformar la segmentació a les coordenades globals de la imatge (estaran en les coordenades del crop), fer servir la funció setFrame().
  • Quedar-se amb la segmentació més gran i filtrar per mida per tal que no trobi cubetes de mida molt petita (poden ser un error del model).
  • Després trobar el contorn i fer buscar el rectangle ortogonal que envolta la segmentació.
  • Mostrar tots els rectangles per pantalla, de color verd.
  • Assigna al rectangles el nom rectCubeta i mostra'ls sota el grup rectCubetes.
  • Crea una Action amb nom mostrarSegmCubetes que mostri opcionalment la segmentació de cubetes en la posició correcta.
  • Assigna a les segmentacions el nom segmCubeta i mostra'ls sota el grup segmCubetes.
tip

Fes servir variables de configuració definides a la seqüència config per assignar colors als Vision Elements, per tal que la combinació de colors sigui més uniforme.

note

Seria millor fer aquesta tasca amb instance segmentation, per trobar cada instància de cubeta i segmentar-la, però per aquest exercici no ho farem. Aquest model està configurat al VisionCloud d'una forma una mica especial, per tal de trobar només l'element que està al centre de la imatge.

✅ Objectius​

  • Aprendre a transformar la posició de les segmentacions (són imatges).
  • Mostrar rectangles al voltant de les cubetes.
  • No mostrar el rectangle si no hi ha cubeta, que no hi hagi falsos positius.
  • Afegir una Action que mostri opcionalment el contingut de les cubetes.

Classificar el contingut de les cubetes​

  • Descarregar el model que troba el contorn de les cubetes. Aquest model es diu af2024_s3_detection_whole i fem servir la versió 3.
  • Importar el model i assignar-lo a un brain amb nom segmFruitsSecs.
  • Aquest model està configurat per treballar amb patches RGB de 700x1000, que es redimensionen per la inferència a 350x500.
  • Executa patch inference individualment a cada cubeta en resolució original de la imatge sobre un crop del rectangle que conté la cubeta.
  • Pel patch inference fer servir context padding de 40px. Veure aquí què és el context padding.
  • El model retorna una segmentació per cada tipus de fruit sec, configurar cada segmentació amb un color diferent segons el tipus de fruit sec segons taula a continuació.
  • Transformar el resultat de la inferència a les coordenades globals de la imatge amb setFrame().
  • Buscar el contorn de cada segmentació i assignar al contorn el mateix color que fruit sec i el nom corresponent.
  • Les segmentacions i contorns s'han d'afegir a la capture en un grup amb nom cubeta{Num}, per tal que els resultats de cada cubeta estiguin en un grup independent.

✅ Objectius​

  • Entendre com funciona el patch inference i el context padding.
  • Mostrar diverses segmentacions en llocs diferents fent servir setFrame().
  • Configurar com es visualitzen les segmentacions (en aquest cas amb colors diferents).

Anàlisi de contaminació i mostra de resultats​

  • Per cada cubeta mostrar en la cantonada superior esquerra els noms dels fruits secs presents, un a sota de l'altre, fent servir el vision element String. Posicionar fent servir setAnchor().
  • Afegir aquests noms al grup per cada cubeta amb nom cubeta{Num}.
  • Inicialment mostrar els textos de color verd.
  • Afegir Actions per tal d'habilitar o deshabilitar l'acceptació de fruits secs a la imatge. Per defecte aquestes Actions per cada fruit sec estaran totes habilitades.
  • Modificar el programa per tal que si en una cubeta hi ha presència d'un fruit sec no acceptat per les Actions (contaminació), es mostri el text d'aquell fruit sec en vermell.
  • Si hi ha contaminació, mostrar el rectangle de la cubeta també en vermell. Això es pot fer modificant la variable del rectangle, encara que s'hagi afegit anteriorment a la capture (funciona per referències, no és una còpia).

✅ Objectius​

  • Lògica avançada fent servir Actions.
  • Afegir text a una Capture.
  • Entendre com funcionen els objectes a OneVision: funcionen per referència com a Javascript.
