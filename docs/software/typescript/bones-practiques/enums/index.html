Enums

No utilitzar enums en Typescript​

Només utilitzar els enums de Typescript quan sigui estrictament necessari (en comptades ocasions!). Són un problema perquè no són tan flexibles ni interpretables com els literals de strings:

// Evitar enums
enum EConnectionStatus {
    Connected = 0,
    Disconnected = 1,
}

// Preferir literals de strings
type TConnectionStatus = "Connected" | "Disconnected";

Aquesta recomanació sorgeix de molts casos en què necessitem convertir un objecte de Typescript a JSON, i els enums no es converteixen a la seva representació string, sinó que es guarden amb el valor numèric. A l'obrir els JSON es fa difícil d'interpretar què significa cada valor numèric, i això pot portar a confusions i errors. Per tant, hem decidit que és millor utilitzar literals de strings en lloc d'enums per garantir una millor interpretabilitat i evitar problemes de conversió a JSON.

Enums en C++

Fins ara els enums de C++ els passàvem "simètricament" a Typescript, però és preferible no mantenir la simetria en aquest cas. Podem tenir enums en C++ i literals de strings a Typescript, sempre que els valors siguin els mateixos.

Per convertir els enums de C++ a literals de string a Typescript, només cal que l'addon converteixi el valor dels enums a string. Per fer això, la recomanació és crear unes funcions auxiliars de conversió: enum_to_string i enum_from_string. Aquestes funcions es poden definir en el mateix header on es defineixen els enums, i ser utiltizades pels addons.

// Convert a tensor datatype to a string.
std::string DECLSPEC_BRAIN_SERVER_API tensor_data_type_to_string(const ETensorDataType data_type) noexcept;

// Convert a string to the corresponding ETensorDataType.
ETensorDataType DECLSPEC_BRAIN_SERVER_API tensor_data_type_from_string(const std::string &data_type_str) noexcept;

Si l'enum a C++ es diu ETensorDataType, s'ha de seguir dient igual a Typescript, encara que sigui fent servir literals de string.
