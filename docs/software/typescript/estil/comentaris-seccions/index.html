Comentaris i seccions

Sobre com fer servir els comentaris i separar seccions en TypeScript.

Seccions​

Utilitzar el següent per separar seccions d’un arxiu. Això permet que es mostri la secció al visualitzador de scrolling de la dreta a VS Code.

// #region Secció correcta
(...)
// #endregion

Això és una convenció: és fàcil i ràpid d’escriure, i és preferible sobre estructures més complexes com:

/////////////////////////
// Secció a evitar
/////////////////////////

Comentaris de funcions​

Fer servir JSDoc (/** */) per documentar funcions. Incloure una descripció breu.

Els paràmetres amb @param i el valor de retorn amb @returns només posar quan calgui, no cal fer-ho sistemàticament perquè hi dediquem massa temps i no aporta gaire valor a nosaltres internament (aportaria valor a un usuari extern).

Sempre acabar les frases dels comentaris amb un punt ".". Això és perquè aquests comentaris es veuen quan poses el ratolí sobre l'element en qüestió a VS Code.

/**
 * Get the available tensor type options from a tensor.
 * Takes into account the tensor data type and shape.
 */
function getTensorTypeOptions(tensor: ITensor): ETensorType[] {
    // ...
}

Quan el comentari és molt llarg, no hi ha problema en separar-lo en diverses línies:

/**
 * The original version from VisionCloud, only used to import the model,
 * although the version cannot be edited by the user, versions in OneVision
 * are determined by the version subfolder, not by this field.
 */
modelVersion: number;

Comentaris de interfaces​

S'ha de posar una descripció general de l'interface i també comentar cada membre per separat. Posar el comentari a sobre de cada membre, no al costat.

/**
 * Information about the state of a brain.
 */
interface IBrainInfo {
    /** The profiler is active for this inference. */
    profilerActive: boolean;

    /** Duration of the last single inference in milliseconds. */
    lastInferenceDuration: number;

    /** Duration of the last patch inference in milliseconds. */
    lastPatchInferenceDuration: number;
}

Comentaris de classes​

Documentar la classe amb una descripció general, i també cada mètode públic i propietat pública per separat.

/**
 * Manages the connection to the inference server.
 */
class InferenceClient {
    /** The current connection status. */
    status: EConnectionStatus;

    /** Send an inference request to the server. */
    async sendInference(image: IImage): Promise<IInferenceResult> {
        // ...
    }
}
