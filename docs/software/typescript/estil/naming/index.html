Naming

Sobre els noms que han de tenir variables, funcions, classes, etc. en TypeScript.

Utilitzar noms curts, però fer no fer reduccions de les paraules, excepte quan el significat de la reducció sigui molt clar: per exemple initCameraSettings().

Classes​

Utilitzar PascalCase per noms de classes. Per exemple: InferenceClient, BrainManager.

class InferenceClient { ... }
class DatasetLoader { ... }

Variables​

Utilitzar camelCase per variables locals, instàncies d'objectes i paràmetres de funcions.

const inferenceResult = await client.sendInference(image);
let currentIndex = 0;

Per membres privats de classes, afegir _ al final del nom:

class InferenceClient {
    private connectionStatus_: EConnectionStatus;
    private reconnect_(): void { ... }
}

Per membres protected, afegir també _ al final del nom:

class BaseHandler {
    protected logger_: ILogger;
}

Funcions​

Utilitzar camelCase per noms de funcions i mètodes. Per exemple: getTensorTypeOptions, calculateDistance.

function analyzeTensor(tensor: ITensor): IAnalysisResult { ... }
function loadConfiguration(): IConfig { ... }

Constants​

Utilitzar SCREAMING_CASE (majúscules amb guions baixos) per les constants. Per exemple:

const MAX_RETRY_COUNT = 3;
const DEFAULT_TIMEOUT_MS = 5000;

Types​

Utilitzar PascalCase amb prefix T per noms de tipus. Per exemple: TGpuConfig, TConnectionStatus.

export type TConnectionStatus = "Connected" | "Disconnected";
Preferir literals a enums

En Typescript preferim sempre literals string a enums, és molt més senzill i flexible de treballar-hi. En el cas de interfícies que vénen de C++, no cal mantenir el mateix nom que ve de C++, es pot canviar a prefix T.

Per exemple, si tenim ELogLevel a C++, a TypeScript es dirà TLogLevel.

Interfaces​

Utilitzar PascalCase amb el prefix I per distingir una interfície d'una classe: IPascalCase. Si per algun motiu no es pot seguir aquesta convenció, no és de vida o mort.

interface IBrainInfo { ... }
interface IInferenceResult { ... }

Per les propietats dels interfaces (o objectes), utilitzar camelCase:

interface IBrainInfo {
    modelName: string;
    version: string;
}
Interícies que vénen de C++

En el cas d'interfícies que vénen de C++ Addons, cal mantenir aquesta convenció, encara que no es respecti la simetria de C++ - Typescript. Per exemple, si a C++ tenim una struct InferenceResult, a TypeScript s'ha de dir IInferenceResult, encara que no segueixi el mateix nom.

Enums​

Utilitzar el prefix E seguit de PascalCase. Els valors de l'enum també en PascalCase.

Els membres de l'enum han d'anar en PascalCase. En el cas de enums que provenen de C++ Addons, han de tenir el mateix nom que vingui de C++, que també hauria de ser en PascalCase.

enum EConnectionStatus {
    Connected = "Connected",
    Disconnected = "Disconnected",
}
