Estil JSON

Sobre els noms que han de tenir els arxius JSON que utilitzem en qualsevol dels packages, i des de qualsevol dels llenguatges (Typescript, C++, Python).

Nom arxius JSON

Preferim utilitzar kebab-case per als noms dels arxius JSON, ja que és una convenció comuna per a fitxers de configuració i dades. Per exemple:

brain-definition.json
camera-settings.json

Contingut

Utilitzar camelCase per tot l'arxiu. Això és necessari perquè moltes vegades convertim els JSON a objectes de Typescript (amb JSON.parse()), on fem servir la convenció camelCase.

note

Antigament havíem fet JSONs amb snake_case, però això ens ha portat a confusions i errors, ja que no és compatible amb la convenció de Typescript. Per exemple, el brain_definition.json era snake_case.
