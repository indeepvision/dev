Entorn C++ per Windows

CMake​

A Windows, descarregar l'instal·lador de CMake de la pàgina web oficial i instal·lar-lo. Instal·lar també el component UI.

Si cmake --version no es troba, potser cal afegir l'executable cmake.exe al $PATH. (Això ho hauria d'haver fet l'instal·lador).

Compilador C++​

Fem servir els Visual Studio Installer, que es pot descarregar des del lloc web de Microsoft descarregant el Visual Studio Community.

caution

Sembla que per defecte s'instal·la el Visual Studio en castellà. Cal instal·lar la versió en anglès perquè és amb la que l'equip està acostumat a veure els errors de compilació.

Es pot instal·lar l'IDE de Visual Studio Community amb la última versió disponible (actualment 2026), després els Build tools ja s'instal·len amb versions específiques.

  • Seleccionar el component principal "Desktop development in C++".

Per poder compilar C++ amb Visual Studio 2017 i 2019, anar a "Modificar/Modify" i instal·lar també els següents components individuals, que es faran servir per compilar les llibreries (MSVC v143 (2022) de moment no es fa servir):

  • MSVC v141 - VS 2017 C++ x64/x86 build tools
  • MSVC v142 - VS 2019 C++ x64/x86 build tools

Ninja​

Trobar l'executable ninja.exe i afegir-lo al $PATH.

A Windows, Ninja s'instal·la juntament amb el compilador MSVC.

  • Si s'ha instal·lat la versió Visual Studio Community, Ninja s'hauria d'haver instal·lat a: C:\Program Files\Microsoft Visual Studio\2022\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\Ninja\.
  • S'ha s'ha instal·lat només els Build Tools (sense la IDE de Visual Studio), Ninja s'hauria d'haver instal·lat a: C:\Program Files\Microsoft Visual Studio\18\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\Ninja\.
tip

Per saber on està instal·lat, executar el terminal especialitzat de Visual Studio x64 Native Tools Command Prompt for VS 2022 i executar where ninja per trobar la ubicació de l'executable. Aquest terminal porta variables d'entorn addicionals per trobar les eines de compilació, que no es troben en el terminal normal de Windows.

OpenCV​

tip

No cal compilar-lo, es pot copiar de companys. Guardar la carpeta d'OpenCV dins de "Program files" amb la versió especificada al nom de la carpeta.

A Windows, la llibreria s'ha de copiar manualment a C:/Program Files/OpenCV <versió>. Crear la variable d'entorn OPENCV_DIR perquè CMake pugui trobar el paquet.

Si es vol compilar, les instruccions són les següents:

Per compilar la llibreria, recordar que s'han de compilar les versions Release i Debug.

cd <carpeta_de_build>
mkdir opencv_4.10_build
cd opencv_4.10_build

Descarregar manualment https://github.com/opencv/opencv/archive/refs/tags/4.10.0.zip i guardar a la carpeta (amb curl a Windows triga massa).

unzip opencv-4.10.0.zip
rm opencv-4.10.0.zip
cd opencv-4.10.0
mkdir build
cd build
cmake -G "Visual Studio 17 2022" -A x64 -T v142 -D CMAKE_BUILD_TYPE=Release -D CMAKE_INSTALL_PREFIX=..\install -D BUILD_EXAMPLES=OFF -D BUILD_opencv_python3=OFF ..
cmake --build . --config Release --target INSTALL

cmake -G "Visual Studio 17 2022" -A x64 -T v142 -D CMAKE_BUILD_TYPE=Debug -D CMAKE_INSTALL_PREFIX=..\install -D BUILD_EXAMPLES=OFF -D BUILD_opencv_python3=OFF ..
cmake --build . --config Debug --target INSTALL

Boost​

tip

A Windows, es pot copiar la llibreria ja compilada de companys per estalviar temps. Guardar la carpeta de Boost dins de "Program files" amb la versió especificada al nom de la carpeta.

Afegir la variable d'entorn BOOST_ROOT perquè CMake pugui trobar el paquet.

Si es vol compilar, les instruccions són les següents:

Clonar el repositori de Boost i compilar-lo amb Bootstrap amb les següents comandes:

.\bootstrap
.\b2 --build-type=minimal address-model=64 link=static

Afegir la variable d'entorn BOOST_ROOT perquè CMake pugui trobar el paquet. Aquestes comandes generaran llibreries estàtiques, canviar-ho segons sigui necessari.

Libzip​

tip

No cal compilar. A Windows i Linux, la llibreria es copia a les carpetes /third-party dels projectes.

De totes maneres, a continuació es descriu com compilar:

Descarregar els binaris compilats de ZLIB (una dependència de libzip) des de http://gnuwin32.sourceforge.net/packages/zlib.htm. Executar la instal·lació. Això copiarà un SDK de ZLIB a C:/Program Files/gnuwin32. Tenir en compte que l'SDK només inclou binaris de 32 bits i necessitem 64 bits. A partir d'aquí, les instruccions són una mica increïbles. Primer de tot, descarregar binaris de 64 bits per substituir els originals de 32 bits. Es poden descarregar des de http://www.winimage.com/zLibDll/. Substituir tant el DLL com el fitxer LIB. Els fitxers zlibwapi.lib i zlibwapi.dll s'han de copiar al directori d'instal·lació de gnuwin32. No cal substituir els binaris de 32 bits.

També cal aplicar un patch a /include/zconf.h com s'explica aquí https://github.com/BioMedIA/MIRTK/issues/142, per un error conegut de ZLIB amb Windows, que normalment implica l'intent d'ús del header de Linux unistd.h. Obrir els fonts amb CMake. Executar configure. Possiblement fallarà perquè CMake no trobi ZLIB. Expandir les opcions clicant la casella "advanced" i establir ZLIB_INCLUDE_DIR i ZLIB_LIBRARY_RELEASE als valors apropiats, apuntant al directori de gnuwin32. NOTA: ZLIB_LIBRARY_RELEASE ha d'apuntar al fitxer DLL zlibwapi.lib i ZLIB_INCLUDE_DIR a ./gnuwin32/include. Executar configure de nou fins que tingui èxit. Deixar la resta d'opcions tal com estan; tot i que no necessitem algunes ara, podríem necessitar-les en el futur. Assegurar-se que ENABLE_SHARED_LIBS està activat. Obrir la solució de Visual Studio i compilar la versió Release.

CUDA​

Instal·lar-lo amb l'instal·lador per defecte descarregat del lloc web de NVIDIA (o ja descarregat per companys).

És la mateixa llibreria CUDA que cal instal·lar per poder fer funcionar el OneVision en producció.

CUDNN​

Descarregar manualment els binaris des d'aquí després d'iniciar sessió: https://developer.nvidia.com/cudnn.

Els binaris i fitxers include s'han de moure a les carpetes d'instal·lació de CUDA.

Tensorflow​

tip

Instal·lar de companys, no compilar des de zero! Instal·lar la carpeta a C:/Program files amb la versió especificada al nom de la carpeta.

Crear la variable d'entorn TF_ROOT apuntant al directori d'instal·lació perquè CMake pugui trobar el paquet.

Si es vol compilar, les instruccions són les següents:

  • Mantenim un fork de Tensorflow amb algunes modificacions. El fork és a l'organització indeepvision a GitHub.
  • Abans de compilar Tensorflow, CUDA, CUDNN i Node.js han d'estar instal·lats.
  • El driver de GPU no s'ha d'instal·lar a WSL (fa servir el driver instal·lat a Windows via virtualització).
  • Després de compilar Tensorflow, crear una variable d'entorn TF_ROOT apuntant al directori d'instal·lació.
  • Aquestes instruccions es basen en https://www.tensorflow.org/install/source_windows (consultar per referència addicional si cal).

Assegurar-se de clonar el repositori des de WSL, en cas contrari els finals de línia de Windows poden causar problemes. Clonar el fork de TF amb:

IMPORTANT: Assegurar-se de fer checkout a la branca correcta!

cd ~
mkdir temp_tf
cd temp_tf
git clone https://github.com/indeepvision/tensorflow.git
git checkout r2.10-Indeep

Instal·lar Python i Pip​

A Windows, instal·lar Python versió 3.6 i seleccionar l'opció d'afegir automàticament les variables PATH necessàries.

Instal·lar paquets de Python​

NOTA: Assegurar-se de fer servir pip3 en lloc de pip.

pip3 install -U --user pip numpy wheel packaging requests opt_einsum
pip3 install -U --user keras_preprocessing --no-deps

Instal·lar Bazelisk amb NPM​

Executar npm install -g @bazel/bazelisk per instal·lar-lo com a dependència global. Les comandes bazel i bazelisk haurien de funcionar ara.

NOTA per a Windows: Cal configurar quin compilador C++ fer servir en cas que hi hagi diversos MS VC++ instal·lats al sistema. Establir la variable d'entorn BAZEL_VC al directori on està instal·lat el toolset VC++, per exemple: C:\Program Files (x86)\Microsoft Visual Studio\2017\Community\VC\. Per VS 2017 i 2019, en cas que hi hagi més d'una versió instal·lada, establir també BAZEL_VC_FULL_VERSION amb la versió completa del compilador. La versió es pot trobar per exemple a C:\Program Files (x86)\Microsoft Visual Studio\2017\Community\VC\Tools\MSVC. Si ja s'ha executat la comanda bazel i la compilació ha fallat perquè el compilador per defecte era incorrecte, cal netejar la configuració amb bazel clean --expunge per eliminar tota la configuració i poder configurar un nou compilador. Assegurar-se de fer servir MS VS 2017 pel compilador.

Si encara apareixen errors de compilació després d'instal·lar un nou compilador, és possible que la compilació faci servir alguna configuració cacheada. Eliminar les dades temporals de Bazel, per exemple a C:/tmp. Per defecte, Bazel farà servir el compilador MSVC més recent.

Instal·lar MSYS​

Instal·lar MSYS des de https://www.msys2.org/ i afegir ~\msys64\usr\bin a la variable d'entorn %PATH, per poder cridar bash des del terminal de Windows.

ALERTA D'ERROR: Afegir MSYS per sobre de C:/Windows/System32 al Path, per evitar que bash.exe de WSL tingui precedència sobre MSYS!

Després d'instal·lar MSYS executar la següent comanda. És important, en cas contrari Bazel no pot descarregar repositoris de tercers.

pacman -S git patch unzip.

Configurar la compilació​

Executar amb el developer command prompt de VS 2019. Per obrir la versió específica de 2019, buscar vcvarsall.bat a la carpeta d'instal·lació de Visual Studio, per exemple: C:\Program Files (x86)\Microsoft Visual Studio\2019\Community\VC\Auxiliary\Build. I després executar el següent des d'un terminal d'administrador (o PowerShell): vcvarsall.bat x64 --vcvars_ver=14.2. 14.2 correspon a MSVC v142, que és el compilador que ve amb Visual Studio 2019.

IMPORTANT: No compilar Windows i Linux al mateix repositori clonat. Per Linux (WSL) clonar a una carpeta separada.

IMPORTANT: S'han detectat errors (tf_to_kernel.cc, MLIR) quan es compila TF amb diferents versions de CUDA disponibles al sistema.

IMPORTANT: Executar amb privilegis d'administrador a Windows! En cas contrari Bazel no pot crear enllaços simbòlics. El servidor de Bazel s'ha de reiniciar després d'obtenir privilegis d'administrador.

AVÍS: No executar la comanda de compilació des del terminal de VSCode, fer servir PowerShell. Especialment a WSL, VSCode es pot desconnectar i deixar el procés de compilació en segon pla.

Executar ./configure des de l'arrel del repositori.

  1. Especificar la ubicació de Python: per defecte normalment funciona, assegurar-se que la comanda python és un àlies de python3
  2. Especificar els paths de la llibreria de Python: per defecte funciona
  3. Voleu compilar TensorFlow amb suport ROCm? No
  4. Voleu compilar TensorFlow amb suport CUDA? Sí
  5. Voleu compilar TensorFlow amb suport TensorRT? No Les ubicacions de CUDA i CUDNN es detectaran automàticament. Comprovar les versions!
  6. Llista de CUDA compute capabilities separades per comes: 5.0, 6.0, 7.5, 8.6, 8.9, 12.0
  7. Flags d'optimització durant la compilació quan s'especifica l'opció de bazel "--config=opt":

    Windows: /arch:AVX /arch:AVX2 /fp:fast

  8. Voleu sobreescriure eigen strong inline per algunes compilacions C++ per reduir el temps de compilació? Sí
  9. Voleu configurar interactivament ./WORKSPACE per a compilacions Android? No

Executar script de compilació​

// Això iniciarà el procés de compilació, que trigarà unes 2-3 hores a finalitzar
bazel build --config=opt //tensorflow/tools/lib_package:libtensorflow --config=cuda

// Fer servir aquesta comanda modificada per limitar els recursos que fa servir Bazel
bazel build --local_ram_resources=HOST_RAM*.4 --local_cpu_resources=HOST_CPUS*.4 --config=opt //tensorflow/tools/lib_package:libtensorflow --config=cuda

// Si cal netejar completament la configuració de compilació
bazel clean --expunge

El procés de compilació crea un fitxer comprimit a bazel-bin/tensorflow/tools/lib_package/ ....

Si la compilació falla amb l'error illegal type for non-type template parameter '__formal executar la comanda de bash de nou per reprendre la compilació (Bazel és incremental). Sembla que és un bug relacionat amb condicions de carrera (veure https://github.com/pytorch/pytorch/issues/25393).

Hi ha errors de compilació al compilar XLA JIT per Tensorflow amb absl::string_view. Cal aplicar un fix a la llibreria abseil com a https://github.com/abseil/abseil-cpp/commit/14550beb3b7b97195e483fb74b5efb906395c31e. TODO: Canviar la versió d'abseil.
