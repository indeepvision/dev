Entorn C++ per Linux

Windows Subsystem for Linux (WSL)​

Fem servir WSL per compilar els binaris de C++ per a Linux.

  • Primer de tot, activar la funcionalitat WSL a Windows. Obrir PowerShell en mode administrador i executar el següent:
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

Importar imatge de Ubuntu a WSL​

// Canviar <user> pel nom de la carpeta d'usuari de Windows
wsl --import Ubuntu-20.04 C:\Users\<user>\AppData\Local\Packages C:\Ubuntu-20.04-v3.tar

Després d'importar, cal establir la distribució com a per defecte (probablement). Per fer-ho:

wsl --set-default Ubuntu-20.04

Després d'importar, l'usuari per defecte (idvision) no estarà establert. Per configurar-lo, anar a la següent clau del registre i trobar la distribució Ubuntu-20.04: HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Lxss

Canviar el valor de la clau DefaultUid a 1000 en decimal (assegurar-se de seleccionar Decimal!). Aquest és l'identificador d'usuari per defecte de l'usuari idvision, el primer usuari creat.

note

Per fer servir el debugger cal que gdb estigui instal·lat a la imatge WSL. També instal·lar p7zip-full amb la comanda sudo apt install p7zip-full.

tip

Les imatges de Ubuntu (WSL) es guarden al PC de test IDVISION-00093P a la carpeta "Documents".

Canviar el nom de la distribució WSL​

Anomenem les distribucions instal·lades segons la versió. Cal que quan es fa wsl --list aparegui el nom de la distribució ben identificat, per no cometre errors:

Ubuntu-20.04-v1
Ubuntu-20.04-v2
Ubuntu-20.04-v3

Això no cal fer-ho, és una nota interna de memòria per si en algun moment cal. Per si després d'importar cal canviar el nom a la distribució:

Get-ChildItem HKCU:\Software\Microsoft\Windows\CurrentVersion\Lxss |
ForEach-Object { Get-ItemProperty $_.PSPath } |
Where-Object { $_.DistributionName -eq "OldName" } |
Set-ItemProperty -Name DistributionName -Value "NewName"

Extensió WSL per VS Code​

  • Des de VS Code, instal·lar l'extensió Remote - WSL. Aquesta extensió facilita treballar a WSL des del VS Code instal·lat a Windows. Ja hauria d'estar instal·lada, perquè està a les extensions recomanades.
  • Un cop instal·lada, es pot canviar entre Windows i WSL amb una comanda des de la paleta de comandes de VS Code: "Reopen folder in WSL" i "Reopen folder in Windows".

Instal·lar manualment la distribució de Ubuntu per WSL​

tip

Això només s'ha de fer per crear una imatge WSL! Normalment només cal importar una imatge ja creada!

  • Instal·lar la distribució de Linux desitjada des de la Windows Store. Per compilar C++ volem Ubuntu 20.04.X. La revisió X pot canviar, fer servir la versió LTS.
  • Probablement caldrà iniciar sessió a la Windows Store.

Un cop instal·lat:

  • Introduir un nou nom d'usuari i contrasenya.
  • Es pot comprovar la versió instal·lada amb lsb_release -a

Exportar una imatge WSL​

Atenció: Això només s'ha de fer per crear una imatge WSL! Probablement només cal importar una imatge ja creada!

Versió d'Ubuntu: Per obtenir la distribució actual, executar lsb_release -a al terminal de WSL.

Exportar la imatge WSL al directori actual amb la següent comanda:

wsl --export Ubuntu-20.04 .\Ubuntu-20.04-v3.tar

Versionem cada imatge de WSL guardada amb un número de versió (si és per ús col·lectiu, si és una cosa especial no cal). Si cal guardar la imatge, guardar-la al PC IDVISION-00093-P C:/Users/idvision/Documents/WSL per tenir-la disponible per a tothom.

Com actualitzar Ubuntu a WSL (de 18.04 a 20.04)​

tip

Això no cal fer-ho, és una nota interna de memòria per si en algun moment cal actualitzar la imatge de WSL que mantenim.

  • Obrir el terminal de WSL i executar les següents comandes:
sudo apt update && sudo apt upgrade -y
sudo apt dist-upgrade -y
sudo apt autoremove -y
  • Després el següent:
sudo apt install update-manager-core
  • Finalment executar l'eina d'actualització de versió (pot trigar una estona):
sudo do-release-upgrade

Atenció: Això només s'ha de fer per crear una imatge WSL! Probablement només cal importar una imatge ja creada!

Usuari i contrasenya per defecte de les imatges WSL: usuari: idvision, contrasenya: idvision.

Per establir una distribució WSL per defecte:

wsl --set-default Ubuntu-20.04

Per obtenir la distribució d'Ubuntu actual des de Linux:

lsb_release -a

glibc​

tip

Això no cal fer-ho si s'ha importat la imatge de Ubuntu amb les dependències preinstal·lades. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

La GNU C Library (no GCC) ja ve instal·lada amb la distribució de Linux, i és millor no tocar-la. És important que quan compilem linkem contra una versió antiga de glibc per tenir compatibilitat cap enrere. Per això fem servir Ubuntu 18.04. Qualsevol versió de Linux més nova que Ubuntu 18.04 també tindrà les versions antigues de glibc instal·lades.

IMPORTANT: Hem provat de fer servir Ubuntu 16.04 però sembla massa antiquat.

Aquesta és una utilitat per comprovar les versions de glibc instal·lades: Navegar a la ubicació de libc.so.6, normalment /lib/x86_64-linux-gnu, i executar el següent: strings libc.so.6 | grep GLIBC.

Driver NVIDIA​

tip

Això no cal fer-ho si s'ha importat la imatge de Ubuntu amb les dependències preinstal·lades. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

Atenció

A WSL 2 la GPU funciona en mode passthrough, per tant es fa servir el driver instal·lat a Windows. El driver de Nvidia s'ha d'instal·lar a Windows, però no a WSL.

Això només funciona a WSL 2, no a WSL 1! L'acceleració GPU no està disponible a WSL 1.

Per comprovar que el driver està disponible des de WSL, executar nvidia-smi al terminal de WSL. Hauria de mostrar la informació de la GPU.

Atenció

Només per WSL, no per Linux natiu.

Informació obtinguda d'aquí: https://developer.nvidia.com/cuda-11-4-4-download-archive?target_os=Linux&target_arch=x86_64&Distribution=WSL-Ubuntu&target_version=2.0&target_type=runfile_local

Linux: Instal·lar CUDA per WSL seguint les instruccions d'aquí: https://docs.nvidia.com/cuda/wsl-user-guide/index.html (canviant la versió de CUDA a la correcta 11.8)

Comandes adaptades per CUDA 11.8:

// Descarregar el fitxer instal·lador SH des de

// Executar l'instal·lador
// Trigarà una estona a mostrar quelcom
sudo sh cuda_11.8.0_520.61.05_linux.run

ATENCIÓ: Desactivar la instal·lació del driver de GPU quan s'instal·la CUDA! Intentar instal·lar el driver a WSL fallarà!

OpenCV​

tip

Això no cal fer-ho si s'ha importat la imatge de Ubuntu amb les dependències preinstal·lades. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

A Linux, les següents comandes descarregaran el paquet d'Internet (compte amb la versió correcta), el descomprimiran, compilaran, instal·laran i afegiran als paths del compilador.

Primer, instal·lar les dependències:

  • CUDA 11.8
  • Instal·lar CUDNN 8.9.6 (o la mateixa versió que fa servir el brain server): Fer servir les instruccions del package brain_server.

Atenció: No descomprimir .tar.gz amb 7z a Windows! Causarà fitxers que falten a la carpeta descomprimida. Fer servir tar a Linux.

// You can run all the commands below in one go:

cd ~
mkdir opencv_build
cd opencv_build
wget https://github.com/opencv/opencv/archive/refs/tags/4.10.0.tar.gz
tar xvfz 4.10.0.tar.gz
rm -rf 4.10.0.tar.gz
cd opencv-4.10.0
mkdir build
cd build
cmake -D CMAKE_BUILD_TYPE=Release -D CMAKE_INSTALL_PREFIX=/usr/local ..
sudo make install

Els includes i llibreries seran trobats per CMake automàticament.

Si g++7 està instal·lat però CMake no el troba, potser no està establert com a compilador per defecte. Configurar-lo amb el següent:

sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-9 100
sudo update-alternatives --install /usr/bin/g++ g++ /usr/bin/g++-9 100

Ara la comanda node -v hauria de retornar la versió instal·lada.

Boost​

tip

Això no cal fer-ho si s'ha importat la imatge de Ubuntu amb les dependències preinstal·lades. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

A Linux, les següents comandes descarregaran el paquet d'Internet (compte amb la versió correcta), el descomprimiran, compilaran, instal·laran i afegiran als paths del compilador.

cd ~
mkdir temp
cd temp
wget https://archives.boost.io/release/1.71.0/source/boost_1_71_0.tar.gz
tar xvfz boost_1_71_0.tar.gz
cd boost_1_71_0
./bootstrap.sh --prefix=/usr/
./b2 --build-type=minimal address-model=64 cxxflags=-fPIC cflags=-fPIC link=static
cd ~/temp
mv boost_1_71_0 ~
cd ~
rm -rf ~/temp

Recordar executar ./b2 clean per recompilar Boost si s'han fet servir paràmetres incorrectes durant la compilació. Després de compilar, afegir la variable d'entorn permanent BOOST_ROOT amb nano ~/.bashrc al principi del fitxer, afegint la línia export BOOST_ROOT="$HOME/boost_1_71_0".

NOTA: Afegim els exports de variables d'entorn al principi del fitxer .bashrc per saltar la part que comprova si la consola és interactiva. Això és perquè quan s'executa wsl echo $BOOST_ROOT des de Windows, la consola no és interactiva i les variables d'entorn del .bashrc no es carreguen. Sempre executem comandes WSL des de Windows amb una primera comanda explícita per carregar les variables d'entorn del .bashrc:

wsl source ~/.bashrc; echo $BOOST_ROOT

Docker​

tip

Això no cal fer-ho si s'ha importat la imatge de Ubuntu amb les dependències preinstal·lades. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

Executar la següent comanda per instal·lar-lo:

sudo apt update
sudo apt install -y apt-transport-https ca-certificates curl software-properties-common
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
apt-cache policy docker-ce
sudo apt install -y docker-ce
sudo usermod -aG docker $USER && newgrp docker

Després d'instal·lar, comprovar la versió amb:

docker -v

Editar el fitxer de configuració de WSL per iniciar Docker automàticament a l'arrencada:

nano ~/.wslconfig

Afegir la següent línia:

[boot]
command="service docker start > /dev/null 2>&1"

Ara instal·lar el CUDA Container Toolkit per poder fer servir la GPU dins de contenidors Docker:

curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/ubuntu20.04/nvidia-docker.list | sudo tee /etc/apt/sources.list.d/nvidia-docker.list
curl -s -L https://nvidia.github.io/nvidia-container-runtime/experimental/ubuntu20.04/nvidia-container-runtime.list | sudo tee /etc/apt/sources.list.d/nvidia-container-runtime.list
sudo apt update
sudo apt-get install -y nvidia-docker2

Configurar el runtime NVIDIA per Docker:

sudo nvidia-ctk runtime configure --runtime=docker

Tancar sessió i tornar a entrar a WSL per reiniciar el servei Docker i que els canvis siguin efectius.

ZBar​

tip

Això no cal fer-ho, està a les carpetes /third-party dels packages que utilitzen la llibreria. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

A Linux, executar el següent per compilar:

Libzip (precompilat i inclòs a third-party)​

tip

Això no cal fer-ho, està a les carpetes /third-party dels packages que utilitzen la llibreria. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

A Linux cal instal·lar primer Zlib: Executar la següent comanda:

sudo apt-get install zlib1g-dev

Després, les següents comandes descarregaran el paquet Libzip d'Internet (compte amb la versió correcta), el descomprimiran, compilaran i instal·laran. Això instal·larà la llibreria en una carpeta temporal, per poder-la copiar a third-party després.

cd ~
mkdir libzip_temp
cd libzip_temp
wget https://github.com/nih-at/libzip/releases/download/rel-1-6-1/libzip-1.6.1.tar.gz
tar xvfz libzip-1.6.1.tar.gz
cd libzip-1.6.1
mkdir build
cd build
cmake .. -DCMAKE_INSTALL_PREFIX=~/libzip_temp/install/
sudo make install

Canviant a -DCMAKE_INSTALL_PREFIX=/usr/local/, s'instal·larà la llibreria a les carpetes estàndard /bin i /include del sistema, i CMake la trobarà automàticament.

Si hi ha problemes executant els tests (no es pot trobar el mòdul libzip.so.5), reiniciar Windows i tornar-ho a provar.

Tensorflow​

tip

Això no cal fer-ho si s'ha importat la imatge de Ubuntu amb les dependències preinstal·lades. Fer-ho només si s'ha instal·lat una imatge de Ubuntu neta.

  • Mantenim un fork de Tensorflow amb algunes modificacions. El fork és a l'organització indeepvision a GitHub.
  • Abans de compilar Tensorflow, CUDA, CUDNN i Node.js han d'estar instal·lats.
  • El driver de GPU no s'ha d'instal·lar a WSL (fa servir el driver instal·lat a Windows via virtualització).
  • Després de compilar Tensorflow, crear una variable d'entorn TF_ROOT apuntant al directori d'instal·lació.
  • Aquestes instruccions es basen en https://www.tensorflow.org/install/source_windows (consultar per referència addicional si cal).

Assegurar-se de clonar el repositori des de WSL, en cas contrari els finals de línia de Windows poden causar problemes. Clonar el fork de TF amb:

IMPORTANT: Assegurar-se de fer checkout a la branca correcta!

cd ~
mkdir temp_tf
cd temp_tf
git clone https://github.com/indeepvision/tensorflow.git
git checkout r2.10-Indeep

Instal·lar Python i Pip​

A Linux, executar el següent:

sudo apt install python3-dev python3-pip

Crear un enllaç simbòlic a python (des de python3) per evitar errors executant bazel:

sudo ln -s /usr/bin/python3 /usr/local/bin/python
sudo ln -s /usr/bin/python3 /usr/bin/python

Instal·lar paquets de Python​

NOTA: Assegurar-se de fer servir pip3 en lloc de pip.

pip3 install -U --user pip numpy wheel packaging requests opt_einsum
pip3 install -U --user keras_preprocessing --no-deps

Instal·lar Bazelisk amb NPM​

Executar npm install -g @bazel/bazelisk per instal·lar-lo com a dependència global. Les comandes bazel i bazelisk haurien de funcionar ara.

Configurar la compilació​

Executar ./configure des de l'arrel del repositori.

  1. Especificar la ubicació de Python: per defecte normalment funciona, assegurar-se que la comanda python és un àlies de python3
  2. Especificar els paths de la llibreria de Python: per defecte funciona
  3. Voleu compilar TensorFlow amb suport ROCm? No
  4. Voleu compilar TensorFlow amb suport CUDA? Sí
  5. Voleu compilar TensorFlow amb suport TensorRT? No Les ubicacions de CUDA i CUDNN es detectaran automàticament. Comprovar les versions!
  6. Llista de CUDA compute capabilities separades per comes: 5.0, 6.0, 7.5, 8.6, 8.9
  7. Voleu fer servir Clang com a compilador CUDA? No
  8. Especifiqueu quin gcc ha de fer servir nvcc com a compilador host: Per defecte hauria d'estar bé
  9. Flags d'optimització durant la compilació quan s'especifica l'opció de bazel "--config=opt":

    Linux: -mavx -mavx2 -mfma -mfpmath=sse -msse4.2 Precaució: Anteriorment fer servir -mfpmath=both produia error de compilació. Veure https://github.com/tensorflow/tensorflow/issues/56639

  10. Voleu configurar interactivament ./WORKSPACE per a compilacions Android? No

Nota si s'afegeixen símbols exportats​

Un fitxer version_script.lds limita els símbols que s'exporten dins l'objecte .so. Fer servir la comanda objdump -TC ./libtensorflow.so per llistar tots els símbols exportats.

Executar script de compilació​

// Això iniciarà el procés de compilació, que trigarà unes 2-3 hores a finalitzar
bazel build --config=opt //tensorflow/tools/lib_package:libtensorflow --config=cuda

// Fer servir aquesta comanda modificada per limitar els recursos que fa servir Bazel
bazel build --local_ram_resources=HOST_RAM*.4 --local_cpu_resources=HOST_CPUS*.4 --config=opt //tensorflow/tools/lib_package:libtensorflow --config=cuda

// Si cal netejar completament la configuració de compilació
bazel clean --expunge
```

El procés de compilació crea un fitxer comprimit a `bazel-bin/tensorflow/tools/lib_package/libtensorflow.tar.gz`.
Descomprimir-lo dins d'una carpeta `tensorflow_installed` amb:
```
tar -xzvf ./libtensorflow.tar.gz
```

Després d'instal·lar, afegir la variable d'entorn permanent **TF_ROOT** amb `nano ~/.bashrc` **al principi del fitxer**, afegint la línia `export TF_ROOT="$HOME/tensorflow_installed"`.
