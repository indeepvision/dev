Inici

En aquesta secció s'explicarà com configurar l'entorn de desenvolupament pas a pas. Depenent del tipus de tasques que hagis de dur a terme, algunes parts te les podràs saltar.

  1. En aquest document — Instal·lar programes, clonar monorepo i configuració general.
  2. Entorn per Typescript — Dependències generals per tots els packages.
  3. Entorn C++ — Versions de dependències i instruccions generals per C++.
  4. Entorn C++ per Windows — Instal·lació de dependències de C++ a Windows.
  5. Entorn C++ per Linux — Configuració de WSL i instal·lació de dependències de C++ a Linux.

VS Code​

Fem servir VS Code per la programació de tota la base de codi: en Typescript, C++ i Pyhton.

Descarregar VS Code de la pàgina oficial. VS Code s'actualitza automàticament, es pot descarregar la última versió.

En cas que una actualització automàtica de VS Code porti problemes (ha passat) es pot fer un rollback.

GitHub Copilot​

Si has de desenvolupar contínuament, demana al teu responsable una llicència de GitHub Copilot per integrar Copilot a VS Code.

Extensions de VS Code​

Les extensions de VS Code recomanades estan a un arxiu extensions.json a indeepvision/.vscode. A l'obrir el monorepo per primera vegada, un diàleg de VS Code demana si instal·lar les extensions recomanades, cal dir que sí.

A continuació alguns comentaris particulars sobre algunes extensions recomanades i la seva utilitat per a un desenvolupador.

Gitlens​

GitLens - Git supercharged. Útil per veure l'historial de cada línia d'un fitxer i tenir informació sobre qui ha fet els canvis.

Compilador Sass (CSS)​

Live Sass Compiler. Necessari per transpilar fitxers SASS a CSS.

note

Mantenir l'arxiu extensions.json només amb les extensions que siguin rellevants.

GitHub Desktop​

Fem servir GitHub Desktop com a client Git visual. Descarregar-lo i inicia sessió amb el teu compte de GitHub.

Clonar monorepo​

Clonar el repositori indeepvision des de GitHub Desktop: File > Clone repository i seleccionar indeepvision/indeepvision. També es pot fer des de la pàgina de GitHub de l'organització.

Escollir una carpeta local sense espais al path, que no estigui dintre de C:/Users, que estigui al C:/. Per exemple crear una carpeta dev a l'arrel del disc dur: C:\dev\indeepvision.

Un cop clonat, obrir la carpeta amb VS Code.

danger

Si es clona el monorepo a una carpeta com "Escriptori" o "Documents", és possible que el OneDrive la sincronitzi, i alentirà molt la càrrega d'arxius. Per això es recomana clonar a una carpeta a C:/.

Els discs durs estan completament encriptats, és prou segur fer-ho així.
