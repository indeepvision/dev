Entorn Typescript

Pel desenvolupament de frontend i backend (no per C++) fan falta molt poques dependències.

En aquest punt assumim que ja tens clonat el monorepo.

Instal·lar Node.js 20.17.0​

Node.js es fa servir per VisionCloud i per tasques auxiliars. No es fa servir per compilar C++ ni tampoc al OneVision (perquè es fa servir Electron, que ja porta Node.js integrat).

Sempre tenim la mateixa versió de Node.js que fem servir amb la versió de Electron de OneVision.

note

Per compilar en C++ contra la llibreria de Node.js, els headers de node es descarreguen automàticament pels scripts de CMake de cada projecte. No cal instal·lar-los per separat.

La llibreria estàtica node.lib també es descarrega.

Instal·lar a Windows​

Per instal·lar Node.js a Windows fer servir NVM (Node Version Manager) per instal·lar la versió correcta. Descarregar de la pàgina de GitHub.

Tancar el terminal i executar nvm ls, aquesta comanda hauria de funcionar i retornar els paquets instal·lats. Per instal·lar una versió específica de node executar nvm install v20.17.0. Ara la comanda node -v hauria de retornar la versió instal·lada.

Per seleccionar la versió correcta de Node.js, executar nvm use v20.17.0, normalment s'ha de fer amb privilegis d'administrador.

Instal·lar a Linux​

Atenció

D'entrada no cal instal·lar Node.js a Linux, fer-ho només quan faci falta. Només podria fer falta si ets un desenvolupador de C++ i has de compilar Vision Elements o si ets un desenvolupador de backend de VisionCloud.

Executar npm install​

Amb el terminal, des de l'arrel del monorepo, executar npm install per instal·lar les dependències de tots els packages.

note

L'arrel del monorepo és la carpeta indeepvision, que és on es troba el fitxer package.json principal. Per exemple, podria ser C:/dev/indeepvision. S'hi podria anar fent cd C:/dev/indeepvision des del terminal.

Les dependències s'instal·len exclusivament a la carpeta indeepvision/node_modules, no s'instal·len a cada package. Això és possible gràcies a la gestió de dependències de Node.js, que permet compartir les dependències entre diferents packages.

Fer build de OneVision o VisionCloud​

note

Fer build a OneVision o VisionCloud vol dir compilar tots els arxius de TS i generar els arxius de JS corresponents. A més a més, compila totes les dependències que tinguin aquests packages i també prepara qualsevol altre cosa que sigui necessària per fer funcionar l'aplicació. Atenció: la comanda de NX build no compila el codi C++, que només es fa quan es fa build-cpp, però que no cal fer perquè els binàris compilats ja es distribueixen amb el monorepo.

Per fer build de OneVision o VisionCloud, executar la tasca de build del package corresponent des de l'eina NX integrada amb Visual Studio Code.

Per exemple, per fer build de OneVision, obrir el package onevision a VS Code i executar la tasca de build fent servir l'eina NX de la barra de menú esquerra.

Aquesta tasca build compila el codi Typescript (no compila C++) del package principal i de totes les dependències que tingui. La configuració de dependències es fa amb NX al package.json de cada package.

note

La tasca build internament fa npm install, de manera que no caldria el pas anterior.

tip

La tasca de NX es pot executar també de la següent manera npx nx run @indeepvision/onevision:build o npx nx run @indeepvision/visioncloud:build.

Si es fa npm run build des del package onevision o visioncloud, també es farà el build, però no es farà automàticament el build de les dependències.

GitHub CLI (gh)​

tip

Això només cal fer-ho si has de pujar releases de OneVision a GitHub, i només a Windows, perquè és l'únic sistema operatiu amb què es puja el release.

Per instal·lar, descarregar la última versió des del lloc web de GitHub i afegir-lo al $PATH si no s'afegeix automàticament.

Una vegada instal·lat, hauria d'estar disponible la comanda gh des de la línia de comandes.

Runtime WIX 3.11​

tip

Això només cal fer-ho si has de pujar releases de OneVision a GitHub, i només a Windows, perquè és l'únic sistema operatiu amb què es puja el release.

electron-wix-msi és un paquet de node que es fa servir per crear instal·ladors de OneVision, a partir de l'aplicació Electron empaquetada.

Necessita el runtime de WIX, que només s'ha d'instal·lar a Windows.

Seguir les instruccions aquí: https://github.com/electron-userland/electron-wix-msi/.

Si no ha canviat res, WIX s'hauria de descarregar des d'aquí: https://wixtoolset.org/docs/wix3/.

ATENCIÓ: Probablement la versió de WIX que cal instal·lar no és la última disponible.

Després d'instal·lar WIX, afegir la següent carpeta al $PATH: C:\Program Files (x86)\WiX Toolset v3.11\bin. (Primer comprovar que WIX s'ha instal·lat aquí). Ara els programes candle i light s'haurien de trobar des de la línia de comandes.
